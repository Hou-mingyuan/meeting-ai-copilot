# 云端实时转写和 AI 答案使用说明

## 怎么用

解压后只需要双击：

```text
启动云端实时转写和AI答案.bat
```

这个脚本会自动完成：

```text
检查 Python
创建 .venv
安装依赖
启动云端实时转写
启动 AI 流式答案
```

启动后打开腾讯会议即可。

## 它会做什么

```text
Windows 系统声音
→ WASAPI Loopback
→ 火山实时语音识别 2.0 WebSocket
→ 实时写入 txt
→ 火山 Coding Plan
→ AI 答案流式写入 txt
```

默认只听电脑播放出来的会议声音，不听本机麦克风。

## 输出在哪里

输出目录：

```text
桌面\实时监听
```

文件名带当天日期：

```text
YYYY-MM-DD_实时监听.txt
YYYY-MM-DD_临时识别.txt
YYYY-MM-DD_AI参考答案.txt
YYYY-MM-DD_运行日志.txt
```

程序跨过凌晨后，会自动写入新日期文件。

说明：

```text
实时监听.txt 只写最终识别结果
临时识别.txt 显示正在变化的 partial 临时结果，会被覆盖更新
AI参考答案.txt 启动时即创建，识别到问题后流式写入答案
```

## 配置 API Key

本开源版**不包含**内置密钥。首次使用前：

```text
copy config.example.json config.json
```

编辑 `config.json` 填入火山引擎凭证，或通过环境变量注入：

```text
实时识别：cloud_asr_api_key / VOLC_ASR_API_KEY
热词表　：cloud_asr_boosting_table_id（可选）
AI 答案 ：ai_api_key / VOLCENGINE_CODING_PLAN_API_KEY
```

不要把含真实 Key 的 config.json 分享给他人或提交到 Git。

## 能用多久（支持时长）

```text
程序本身不限时长：跨天自动换文件、断网自动重连，可以一直开着，一整天面试也没问题。
真正的上限是火山账号这个 Key 的「时长包」余额——识别用的是小时版
（volc.seedasr.sauc.duration），按音频时长计费。
还能用多少小时，去火山控制台「用量统计」查。一场面试一般 1~2 小时，时长包够就行。
连接断了会自动重连续上，使用时基本感觉不到中断。
```

## 专业词（热词）

已接入火山控制台「热词表」，对全栈面试高频词（事务、并发、集合、索引、缓存、多线程……）做了识别优化，提升准确率。

```text
词表位置：火山控制台 → 自学习平台 → 热词管理
引用方式：config.json 的 cloud_asr_boosting_table_id 填该词表 ID
加词办法：直接在控制台那个词表里加，保存即生效，包不用重打
没配词表：会退回用 config.json 里的 cloud_asr_hotwords（内联热词，上限约 100 字）
```

注意：词表必须和 ASR Key 在同一个应用下才生效。

## 出问题发回什么

发回：

```text
桌面\实时监听\YYYY-MM-DD_运行日志.txt
config.json
```

如果没有生成文件，先确认腾讯会议声音确实从电脑耳机或扬声器里播放出来。
